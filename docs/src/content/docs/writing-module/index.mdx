## Module Structure

A module is a standard **[Laravel Service Provider](https://laravel.com/docs/providers)** that additionally implements the
[`Epsicube\Support\Contracts\IsModule`](https://github.com/epsicube/framework/blob/main/packages/Foundation/Support/IsModule.php) interface.

This contract exposes the metadata required by Epsicube to register, identify, and manage your module.

## How to Create a Module?
<br/>
<Steps>
1. ### Create a Service Provider

   Generate a new module using Laravel’s CLI:

   ```bash
   php artisan make:module MyCustomModule
   ```

    <Aside type="note">
    This class forms the core of your module.
    </Aside>

2. ### Define Metadata

   Open the generated file `app/Modules/MyCustomModule.php`

   ```php
   use Illuminate\Support\ServiceProvider;
   use Epsicube\Support\Contracts\IsModule;
   use Epsicube\Support\Modules\Identity;
   use Epsicube\Support\Modules\Module;

   class MyCustomModule extends ServiceProvider implements IsModule
   {
       public function module(): Module
       {
           return Module::make('custom::your-module-identifier', '0.0.1')
               ->providers(static::class)
               ->identity(fn (Identity $identity) => $identity
                   ->name('MyCustomModule')
                   ->description('')
                   ->author('Internal dev')
               );
       }

       //...
   }
   ```

    <Aside type="caution">
    Ensure that your <strong>identifier</strong> is globally unique within the application.
    </Aside>

3. ### Register the Module

   By default, the `module:make` command registers your module in `bootstrap/modules.php`.

   You may also manually create or update the `bootstrap/modules.php` file:

   ```php
   <?php

   return [
       //...
       App\Modules\MyCustomModule::class,
   ];
   ```

4. ### Enable the Module

   By default, the `module:make` command enables your module (unless you use the `--disabled` option).

   You may also enable it manually using:

   ```bash
   php artisan modules:enable {identifier}
   ```

</Steps>
